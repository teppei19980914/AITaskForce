## 2. 【最重要】絶対にやってはいけないこと（禁止事項）

### 2.1 入力してはいけない情報

以下の情報は **チャット・音声・ファイルアップロードを問わず一切入力禁止**。

| 分類 | 具体例 | リスク |
|---|---|---|
| **顧客情報** | 氏名・連絡先・契約金額・システム仕様・個人情報 | 個人情報保護法・守秘義務違反 |
| **案件固有情報** | 顧客名が特定できる案件名・PJID・要件定義書・設計書 | 機密情報漏洩 |
| **グループ財務情報** | 未公表の売上・利益・予算・計画 | インサイダー情報漏洩 |
| **人事情報** | 評価・処遇・給与・採用選考の情報 | プライバシー侵害 |
| **投資・提携情報** | 検討中の投資・M&A・事業提携 | 機密漏洩・インサイダー |
| **認証情報** | パスワード・APIキー・トークン・秘密鍵・証明書 | セキュリティインシデント |

> **判断に迷ったら入力しない。** 代わりに匿名化・抽象化（「顧客A」「金額X円」）して質問する。
> 顧客から書面で承認されたものは入力可。

### 2.2 やってはいけない行為

- **対外資料への無断転用** — 確認なしに顧客向け資料・提案書に流用しない
- **ファクトチェックなしの使用** — 数値・統計・事実をそのまま信用しない（ハルシネーション）
- **Coworkへの過度な権限付与** — 機密フォルダへのアクセスは付与しない
- **機密システムへのChrome操作** — 社内管理・人事・財務システムでは使用禁止
- **外部ファイルの無警戒使用** — ベンダー・顧客から受け取ったファイルをExcel/PPTで読まない（プロンプトインジェクション攻撃のリスク）
- **生成物の著作権無視** — 商用利用前に既存著作物との照合が必要
- **System Promptの無断変更** — Admin/TF責任者以外は変更禁止
- **機密文書のProject知識ベース追加** — Project利用者全員がアクセスできる状態になる

## 3. Claudeの特性として理解すべきこと

| 特性 | 内容 | 対処 |
|---|---|---|
| **ハルシネーション** | 存在しない事実・誤った数値・架空の引用を自信を持って提示する | 重要情報は必ず一次情報で確認。〔要確認〕付きは確認前に使用しない |
| **知識の時間的限界** | 学習データは2025年8月まで。法改正・製品仕様の最新情報は非反映 | Web検索を有効化、または一次情報で別途確認 |
| **最終責任は使用者** | 出力はあくまで参考情報。判断と責任は使用者が負う | Claudeは「素材と判断補助のツール」と位置づける |

## 4. ツール別の注意事項

### 必須ツール一覧

| ツール | 状態 | 主用途 |
|---|---|---|
| Claude Desktop（Win/Mac） | 正式リリース | 全活用の起点 |
| Cowork | Research Preview | ローカルファイル操作・自動化 |
| Claude for Excel | Beta | 財務モデル分析・データ整形 |
| Claude for PowerPoint | Beta | スライド生成・編集 |
| Claude in Chrome | Experimental | Web情報収集・調査支援 |
| MS365 Connector | 必須 | SharePoint・Teams連携 |
| Claude iOS/Android アプリ | 必須 | モバイル利用・音声入力 |

### Cowork（Research Preview段階）の注意

- **監査ログ非記録** — 業務エビデンスが必要な作業には使用しない
- **実行前の確認** — 「承認してから考える」ではなく「確認してから承認する」
- **権限の最小化** — 機密フォルダへのアクセス権は付与しない
- **会話履歴はローカル保存** — Anthropicのデータ保持ポリシー対象外

### Claude for Excel / PowerPoint（Beta段階）の注意

- **プロンプトインジェクション** — 外部から受け取ったファイルには使用しない（自社作成ファイルのみ）
- **データ自動削除** — 入力・出力は30日以内に自動削除。重要な中間成果物はローカル保存
- **クロスアプリ連携可能** — Excel ⇔ PowerPoint でコンテキスト共有可

### Claude in Chrome（Experimental段階）の注意

- **意図しない操作のリスク** — フォーム送信・クリック・ページ遷移
- **使用範囲を限定** — 機密性の高いシステム（社内管理・人事・財務）では使用禁止
- **公開情報の収集・調査用途に限定**

### 音声入力の注意

- **Voice Mode は英語のみ対応**（2026年4月時点）— 日本語の双方向音声会話は未対応
- **日本語音声入力**はモバイルのディクテーション機能 or OS標準機能を使用
- **公共の場での使用禁止** — 周囲に聞こえる可能性。個室・自席で使用
- **サーバー送信される** — 機密情報の音声入力もテキストと同じ基準で禁止

### MS365 Connector の注意

- **接続範囲を必要最小限に** — 不要に広く接続しない。定期的に見直し
- **クロステナント制約** — softec-ic と lt-s.jp の異なるテナント間では接続できない場合あり

## 5. 3つのProjectの役割と使い分け

| Project名 | 役割 | 判断基準 | こんなときに使う |
|---|---|---|---|
| **TF_Base** | 全会話の前提・日常業務実行支援 | 「知らないと困る」常識・前提情報 | 通常の業務支援、議事録作成、文書作成 |
| **TF_Prompt_Library** | プロンプト資産の蓄積・管理・改善 | 「使える」資産・TFの活動成果 | プロンプトを作りたい・改善したい |
| **TF_Research** | 調査・収集・分析・ナレッジ蓄積 | 「必要なときに掘る」参照情報 | 調査・分析・社内ナレッジ参照 |

### Project間の誘導フロー

```
TF_Base（行動規範の基準）
  ├─ 日常業務の実行支援
  ├─ 「プロンプトを作りたい」 → TF_Prompt_Library へ
  └─ 「調査・分析したい」     → TF_Research へ

TF_Research
  └─ 調査結果を成果物にしたい → TF_Base へ（ブリーフィングメモ経由）
```

### Projectに関する重要事項

- **Projectのチャットはプライベート** — Publicでも他メンバーには見えない
- **知識ベース追加は要注意** — 追加した文書はProject利用者全員がアクセス可能。機密レベルを必ず確認

## 6. 生成物の品質確認チェックリスト

対外資料・顧客提案・社内重要文書に使用する前に必ず確認すること。

- [ ] **ファクト確認** — 数値・統計・固有名詞・法的情報は一次情報で確認済みか
- [ ] **機密情報の混入** — 顧客名・案件情報等が含まれていないか
- [ ] **著作権確認** — 既存著作物からの無断引用・類似表現がないか
- [ ] **最新情報の確認** — 法改正・製品仕様・市況等が正確に反映されているか
- [ ] **論理確認** — 生成物の論理構成・結論が業務判断として妥当か
- [ ] **対外使用の承認** — 顧客提案・対外公開資料は上長の確認・承認を得ているか

---

## 7. インシデント発生時の対応

1. **即時報告** — 違反・漏洩の可能性を認識したら直ちにTF責任者に報告
2. **証跡の保全** — スクリーンショット等を保存（Coworkを除く）
3. **使用の一時停止** — 調査中は該当ツールの使用を停止
4. **再発防止** — TF責任者が原因分析、必要に応じて文書改訂
