# AI駆動開発 vs 従来開発 比較レポート

## 概要

本レポートは、ユメログ（YumeLog）アプリ開発の実績データをもとに、**同一条件の開発者**が「従来の人間主体開発」と「AI駆動開発」を行った場合の定量比較を行う。

AI駆動開発の実績値はプロジェクトの Git 履歴・ソースコード統計から取得した確定値であり、従来開発の推定値は業界標準の生産性指標と開発者の経験レベルに基づく算出値である。

|---|---|---|---|---|

| 開発 | 14日 | 3〜6ヶ月 | **85〜95%** | コード生成・テスト生成・リファクタリング |

## 定量比較表

### 1. 開発期間・工数

| 項目 | AI駆動開発（実績） | 従来開発（推定） | 倍率 | 推定根拠 |
|---|---|---|---|---|
| 開発期間 | **29日** | 6〜9ヶ月 | **6〜9倍短縮** | IPA SEC「ソフトウェア開発データ白書」中小規模（5万行）の中央値。未経験フレームワークの学習期間1〜2ヶ月を含む |
| 実稼働日数 | **21日** | 120〜180日 | **6〜9倍短縮** | 上記期間を営業日換算 |
| 1日あたり生産行数 | **2,281行/日** | 50〜100行/日 | **23〜46倍** | IPA SEC白書：新規開発の生産性中央値は約50〜100行/人日（テスト込み）。未経験技術では下限寄り |
| 学習コスト | **0日（開発と同時）** | 1〜2ヶ月 | **省略** | Flutter/Dart/Drift/Riverpod等の学習。スクラッチ開発経験なしの開発者にとって、アーキテクチャ設計の習得が必要 |

**推定根拠の詳細:**

- IPA（独立行政法人 情報処理推進機構）「ソフトウェア開発データ白書」2022-2023によると、新規開発のFP（ファンクションポイント）あたり生産性は、中小規模プロジェクトで 10〜20 FP/人月が中央値
- 本プロジェクトの機能規模（32機能・6テーブル・8画面）は概算で 200〜300 FP に相当
- 200 FP / 15 FP/人月 = 約13人月 → 1人で約13ヶ月が統計的中央値だが、Web単一プラットフォームであること、デザイン工程の簡略化を考慮し 6〜9ヶ月と推定
- 未経験フレームワーク（Flutter）での開発は、経験ありの場合と比較して生産性が 40〜60%低下する傾向がある（出典: COCOMO II モデルの PLEX 係数）

## 定性比較

### AI駆動開発で特に効果が大きかった領域

| 領域 | 効果 | 具体例 |
|---|---|---|
| **未経験技術の即時活用** | 学習と実装を同時に実行。学習期間ゼロ | Flutter/Dart/Drift/Riverpod/Firebase を初日から使用 |
| **設計判断の高速化** | アーキテクチャ選定・設計パターン適用が即座 | 3層アーキテクチャ、Provider設計、ORM選定をAIが提案・実装 |
| **テスト自動生成** | テスト作成の心理的障壁がゼロ | 813テストケースが開発と同時に生成 |
| **横展開・リファクタリング** | 同一パターンの網羅的な修正が確実 | 750ラベルの外部化、全SnackBarの統一化 |
| **ドキュメント整備** | 実装と同時に仕様書・設計書を生成 | 7ドキュメントが開発期間内に完成 |
| **セキュリティ・品質** | ベストプラクティスが自動適用 | XSS対策、入力バリデーション、N+1クエリ防止 |
| **CI/CD構築** | インフラ知識なしでパイプライン構築 | GitHub Actions でテスト・ビルド・デプロイの完全自動化 |

### AI駆動開発の課題・限界

| 課題 | 具体例 |
|---|---|
| **意思決定は人間が必要** | 「活動予定」という名称の採用、UIレイアウトの最終判断は開発者が実施 |
| **ドメイン知識の注入** | 「夢→目標→タスク」の階層構造、ゲーミフィケーション設計は開発者のビジョンに依存 |
| **外部サービス連携のデバッグ** | Stripe Customer Portal の CORS 問題、Firebase Auth の Windows ビルド問題は試行錯誤が必要 |
| **モバイル実機での確認** | UIの視認性・操作性はAIでは判断できず、人間によるテストが不可欠 |

## 結論

AI駆動開発は、特に**未経験技術での新規スクラッチ開発**において最も大きな効果を発揮する。本プロジェクトでは、Flutter/Dart 未経験の開発者が、21日間の実稼働で**約4.8万行・813テストケース・32機能**を備えたプロダクション品質のアプリを完成させた。

最大の効果は「学習コストの消滅」と「品質基盤の自動構築」にある。従来であれば1〜2ヶ月を要するフレームワーク学習が不要となり、さらにテスト・CI/CD・セキュリティ対策・ドキュメントといった「やるべきだが後回しにされがちな品質基盤」がAIにより開発と同時に構築された。

一方、プロダクトのビジョン・ドメイン設計・UX判断は引き続き人間の役割であり、AIは「人間の意思決定を高速に具現化するエンジン」として機能する。
